# 生产环境 etcd 如何避免单点问题？

1. 客户端（包括 etcdctl 和业务程序）应该通过域名服务（DNS）查找 etcd 可用实例，避免直接写目标 IP 导致单点问题。

# DNS 如何避免单点问题？

- Anycast（世界级大场景）：通过运营商（BGP 能力）支持划区域部署同 IP 服务，自动路由到最近的节点，如 8.8.8.8 在全球有用 200+ 个数据中心。
- 局域网 VIP（企业级）：每个虚拟 IP 背后都对应了一个集群，集群通过 KeepAlived 模式抢占 VIP，确保一直有节点提供服务。
- 主备服务器（小范围）：操作系统的 ``/etc/resolv.conf`` 支持填写多个 nameserver，只有所有 nameserver 都挂了才算真挂。

# DNS 主备服务器方案如何在客户端不感知的情况下更换 DNS 服务器？

1. 运维在 DHCP 服务器（通常是路由器）修改 DNS Server。
2. DHCP 协议会在客户端启动及续租时，下发给客户端“客户端 IP”，“网关”，“**DNS 服务器**”，“租期”。

# 怎么避免路由器单点故障？

- 硬件层面部署多台路由器，且保持相同的虚拟网关 IP。
